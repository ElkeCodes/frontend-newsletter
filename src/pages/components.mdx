**Bold text** stands out from the paragraph. *Italic text* adds soft emphasis. You can also combine them for ***bold italic***.

## Code

### Inline code

Use `npm run build` to compile the newsletter. The `<NewsletterLayout>` component lives in `src/layouts/`.

### Code block

```typescript
interface NewsletterFrontmatter {
  title: string;
  subject?: string;
  date: string;
  author?: string;
  preview?: string;
}

async function renderNewsletter(slug: string): Promise<string> {
  const content = await import(`../pages/newsletters/${slug}.mdx`);
  return content.default;
}
```

### What shipped this week

We landed three improvements in the last sprint:

1. **Link preview cards** — drop any URL into an MDX file with `<LinkPreview url="..." />` and a card is fetched and rendered at build time.
2. **Dark mode palette** — all CSS variables now have proper dark-mode overrides, including `--link-color-hover`.
3. **Heading hierarchy** — h2 through h4 now have distinct sizes, weights, and treatments.

> These changes are all backwards-compatible. Existing `.md` newsletters render without modification.

The next sprint focuses on:

- Outlook-safe inline CSS output
- A send script using Microsoft Graph
- Recipient list management
